# JARVIS DASHBOARD MOCKUP

## Action Center / Dashboard Concept

The dashboard replaces the standard empty screen with a highly functional summary block that dynamically updates based on the time of day and the system's global state.

```text
+-------------------------------------------------------------+
|                                                             |
|  Good Morning Boss                                          |
|                                                             |
|  [ Current Focus ]                                          |
|  Jarvis OS Refactoring                                      |
|                                                             |
|  [ Active Project ]                                         |
|  Jarvis Dashboard Implementation                            |
|                                                             |
|  [ Current Workspace ]                                      |
|  Jarvis (Suggested: Vision)                                 |
|                                                             |
|  [ Digital Activity ]                                       |
|  Activity: Building | Focus Drift: None                     |
|                                                             |
|  [ Quick Actions ]                                          |
|  [ Continue Session ] [ Show Pending ] [ Morning Brief ]    |
|  [ Open VS Code ] [ Refresh Dashboard ]                     |
|                                                             |
|  [ Status ]                                                 |
|  CPU: 14% | RAM: 45% | OS: Windows | Pending Tasks: 3       |
|                                                             |
+-------------------------------------------------------------+
```

### Time of Day Variations
- **05:00 - 11:59**: "Good Morning Boss" (Includes `Morning Brief` action)
- **12:00 - 16:59**: "Good Afternoon Boss" (Includes `Afternoon Review` action)
- **17:00 - 04:59**: "Good Evening Boss" (Includes `Evening Summary` action)

## Performance Rules
- **No LLM required**: The text is strictly generated by checking `datetime.now()` and reading `jarvis_os/core/global_state.py`.
- **Render Speed**: The `/dashboard` endpoint evaluates pure Python dictionaries and returns in <10ms. 
- **Caching**: The frontend Javascript utilizes a 30-second memory cache to prevent spamming the backend API if the user rapidly refreshes the UI or toggles panels.
