# JARVIS MANIFESTO

JARVIS is a personal AI Operating System.
It is NOT a chatbot.
It is NOT AGI.
It is NOT an autonomous agent.
It exists to reduce Boss's cognitive load.

## The 5 Pillars
Every feature must satisfy at least one purpose:
1. Memory
2. Awareness
3. Organization
4. Friction Reduction
5. Actions

## The 3 Tests
Every feature must pass three tests:
1. Can Boss see it?
2. Can Boss demo it?
3. Will Boss use it tomorrow?

## System Architecture Rules
- No duplicate brains.
- No duplicate memory systems.
- No duplicate suggestion systems.
- No new folders without long-term justification.
- The product must become smaller over time, not larger.

## The Ultimate Goal
Observe
↓
Understand
↓
Guide
↓
Reduce Friction
↓
Act

## The 80/20 Rule
- 80% of development effort must improve existing capabilities.
- 20% of development effort may introduce new capabilities.
- JARVIS should evolve by becoming simpler, faster, and more useful rather than larger.
- If a feature can be implemented inside an existing module, do not create a new folder.

## Architecture Freeze
The architecture is permanently frozen to the following modules. **No more folders.**
- `jarvis_os/core/`
- `jarvis_os/dashboard/`
- `jarvis_os/observers/`
- `jarvis_os/operator/`
- `jarvis_os/security/`
- `jarvis_os/session/`
- `jarvis_os/workspaces/`
- `jarvis_os/desktop_action/`

## The Founder Workflow
Track friction, not vanity metrics.
1. Use Jarvis 30 minutes daily.
2. Write 3 annoyances (Friction Log).
3. Fix them.
4. Repeat.

## Android Companion Rules
- Android never owns business logic.
- Android never creates intelligence.
- Android never stores memory.
- Android only consumes existing endpoints.
- If Android can be deleted without affecting JARVIS, the architecture is correct.

## Telegram Bridge Rules
- Telegram owns zero business logic.
- Telegram owns zero memory.
- Telegram owns zero intelligence.
- Telegram owns zero state.
- Telegram only translates messages into existing actions.
- If Telegram is deleted tomorrow, JARVIS must continue working perfectly.
